# How to create a multi-transfer "distribute" transaction with Shardus

Now that you're familiar with the shardus-cli-tool and running local networks, you can start creating different kinds of transactions within your applciation.

  1. Install project dependencies

npm install

  1. Start the network with a single node

npm run start

  1. Create a distribute command in client.js
vorpal
  .command(
    "tokens distribute <amount> <source> [recipients...]",
    "Transfers <amount> tokens from the <source> wallet to each of the [recipients] wallets."
  )
  .action(function(args, callback) {
    const sourceId = walletEntries[args.source];
    const recipientIds = args.recipients.map(name => walletEntries[name]);
    injectTx({
      type: "distribute",
      from: sourceId,
      recipients: recipientIds,
      amount: args.amount
    }).then(res => {
      this.log(res);
      callback();
    });
  });
  1. Within the validateTransaction transaction function of dapp.setup, add a case to the switch statement for the "distribute" transaction type. (Make sure the result gets marked "pass")
case  "distribute":
  response.result  =  "pass";
  response.reason  =  "This transaction is valid!";
  return  response;
  1. Within the apply function of dapp.setup, add a case to the switch statement for the "distribute" transaction type.
case  "distribute": {
  const from = wrappedStates[tx.from].data;
  const recipients = tx.recipients.map(recipientId  => wrappedStates[recipientId].data);
  // Ensure that the sender has enough balance to cover the entire distribution
  if (from.data.balance < recipients.length * tx.amount) {
    throw  new  Error(`account ${tx.from} has insufficient funds to cover this transaction`);
  } else {
    recipients.forEach(recipient  => {
      from.data.balance -= tx.amount;
      recipient.data.balance += tx.amount;
    });
    console.log("successfully applied distribute transaction");
  }
  break;
}
  1. Within the getKeyFromTransaction function in dapp.setup, add a case to the switch statement for the "distribute" transaction type
case  "distribute":
  // tx.recipients is an array of the target keys
  result.targetKeys  = tx.recipients;
  result.sourceKeys  = [tx.from];
  break;
  1. You can create multiple wallets with tokens using the following commands:
tokens create 100 myWallet
tokens create 100 myWallet2
tokens create 100 myWallet3
tokens create 100 myWallet4
  1. Test your new distribute transaction (sends 5 tokens to 3 wallets in one transaction)

tokens distribute 5 myWallet myWallet2 myWallet3 myWallet4
