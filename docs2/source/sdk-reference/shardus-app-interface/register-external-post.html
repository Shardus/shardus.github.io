# registerExternalPost

shardus exposes a method called registerExternalPost, which allows you to register POST http endpoints on your server. Currently, this is really only useful for creating the POST endpoint for injecting transactions. Since POST requests often modify or add state to an application, we can't exactly do that sort of thing outside of the apply function, or the nodes will become out of sync.

This is how registerExternalPost is being used in our applications currently. Within the POST endpoint for the application, we need call another method exposed by shardus called put, which essentially sends the tranaction through to be processed by your application. Another thing to note, is that put expects an object named "tx". In this case, the client has put the "tx" object into the request body.

const dapp = shardus();

dapp.registerExternalPost("inject", async (req, res) => {
  const result = dapp.put(req.body);
  res.json({ result });
});
